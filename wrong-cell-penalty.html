Wrong-Cell Penalty

wcpenalty adds a penalty term that pushes down the model's probability on cells it predicts wrong. Investigation of the derive_ignore + wcpenalty stack:
• The original -log(1-p) form has an unbounded gradient at p→1 — rare but ×100–×1000 spikes that poison Adam's v.
• Fix: soft cap (tanh, bounded gradient) + per-cell-fair normalization — CE and wcpenalty share the same divisor so each wrong cell is weighted exactly like each CE cell. λ=0.15.
• Result: V1 (÷56) and V2 (÷81) reach 0.742 / 0.735 all-exact @ nsup64, vs CLEAN peak 0.697. Caveat: CLEAN is its peak ckpt (step 41664); the wcpenalty runs are step 65100 — not the same step, so not a strict comparison.
• Lowering both thresholds to 0.90 (last row) hurts badly (−13pp @ nsup64).
Metric: exact_acc_at_halt_or_max, FULL sudoku-extreme test set (422,786 puzzles). Cell colour = vs CLEAN baseline (green better, red worse). Sorted by hard@nsup64.
Formulas
Total training objective: \[ \mathcal{L}_{\text{total}} \;=\; \mathcal{L}_{\text{CE-DI}} \;+\; \lambda\,\mathcal{L}_{\text{wc}} \;+\; \tfrac{1}{2}\bigl(\mathcal{L}_{q\text{-halt}}+\mathcal{L}_{q\text{-cont}}\bigr) \] wcpenalty term, two forms — \(p\) = model softmax prob on its (wrong) predicted class, penalty averaged over the per-sample divisor \(D\): \[ \mathcal{L}_{\text{wc}}^{\text{neglogp}}=\frac{1}{D}\sum_{i}-\log(1-p_i) \qquad \mathcal{L}_{\text{wc}}^{\text{focal}}=\frac{1}{D}\sum_{i}p_i^{\gamma} \] The \(-\log(1-p)\) form has unbounded gradient as \(p\to1\). Soft cap bounds it: \[ \Phi(p)=C\,\tanh\!\Bigl(\tfrac{-\log(1-p)}{C}\Bigr),\qquad C=2 \] Per-cell-fair normalization: CE-DI and wcpenalty use the same divisor \(D\) (supervised-cell count for ÷56, or all-81 for ÷81), so each wrong cell is weighted exactly like each CE cell: \[ \mathcal{L}_{\text{CE-DI}}=\frac{1}{D}\sum_{i\in\text{sup}}\text{CE}_i \qquad \mathcal{L}_{\text{wc}}=\frac{1}{D}\sum_{i\in\text{wrong}}\Phi(p_i) \]
